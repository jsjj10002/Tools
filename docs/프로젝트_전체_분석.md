## 프로젝트 개요

### 프로젝트 목적
개인적으로 자주 사용하는 다양한 도구들(파일 처리, 이미지 편집, 영상 변환 등)을 웹 환경에서 통합하여 제공하는 **Progressive Web App (PWA)**입니다.

### 핵심 특징
- **오프라인 우선**: 인터넷 연결 없이도 대부분 기능 사용 가능
- **클라이언트 사이드 처리**: 모든 파일 처리가 브라우저 내에서 로컬로 진행되어 개인정보 보호
- **PWA 지원**: 데스크톱과 모바일에서 앱처럼 설치 및 사용 가능
- **실시간 진행률 추적**: 모든 작업의 상세한 진행 상황 실시간 표시

### 개발 현황
- **시작일**: 2024년 6월 19일
- **중단 기간**: 2024년 12월 22일 ~ 2025년 6월 18일 (약 6개월)
- **재시작일**: 2025년 6월 19일
- **현재 단계**: Phase 2 진행 중 (약 70% 완료)
- **전체 진행률**: 약 50-70%

## 아키텍처 분석

### 전체 아키텍처

```
┌─────────────────────────────────────┐
│      Presentation Layer             │
│  (React Components, Pages)          │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      State Management Layer         │
│  (Zustand Stores: app, tool, task) │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      Business Logic Layer           │
│  (Services: PDF, Image, Video)     │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      Browser API Layer              │
│  (File API, Canvas, Web Workers)    │
└─────────────────────────────────────┘
```

### 데이터 흐름

1. **사용자 입력** → 컴포넌트
2. **컴포넌트** → 상태 관리 (Zustand)
3. **상태 관리** → 서비스 레이어
4. **서비스 레이어** → 브라우저 API
5. **결과** → 상태 업데이트 → UI 업데이트

### 작업 관리 시스템

**구현 상태:** ✅ 완성

**특징:**
- 작업 추가/업데이트/제거 기능
- 실시간 진행률 추적
- 작업 완료 시 자동 정리 (2초 후)
- 다중 작업 동시 처리 지원

**아키텍처 평가:**
- ✅ 관심사 분리가 잘 되어 있음
- ✅ 상태 관리가 명확함
- ⚠️ 서비스 레이어가 일부 도구에만 존재 (일관성 부족)
